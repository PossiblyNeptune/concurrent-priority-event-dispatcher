Dynamic Aging Algorithm

Prevents low-priority events from starving under heavy workloads. Once an event's queue duration exceeds 50ms (simulated as 5000ms), it's promoted to the next priority level.

O(1) Efficiency: Because lanes are FIFO, the oldest element is always at the front. We only check the front elements, avoiding binary heap rebuild costs.

// event_bus.cpp
void EventBus::apply_aging_under_lock(uint64_t now_ns) {
    while (!med_queue_.empty()) {
        if (now_ns - med_queue_.front().last_promoted_ns > AGING_THRESHOLD_NS) {
            Event ev = med_queue_.front(); med_queue_.pop();
            ev.priority = Priority::HIGH;
            ev.last_promoted_ns = now_ns;
            high_queue_.push(ev);
        } else break;
    }
    while (!low_queue_.empty()) {
        if (now_ns - low_queue_.front().last_promoted_ns > AGING_THRESHOLD_NS) {
            Event ev = low_queue_.front(); low_queue_.pop();
            ev.priority = Priority::MEDIUM;
            ev.last_promoted_ns = now_ns;
            med_queue_.push(ev);
        } else break;
    }
}

Bounded Capacity & Backpressure

Restricts queue capacity (max 15 items in visualizer, 50 in C++) to prevent memory explosion. If full:

  1. If a `LOW` priority AI event exists, it is popped and dropped to immediately accommodate incoming telemetry (Lossy policy).
  2. Otherwise, the pushing thread blocks via `not_full_cv_.wait()` until a worker consumes an event and frees space (Backpressure).
// event_bus.cpp
void EventBus::push(Event event) {
    std::unique_lock lock(mutex_);
    while (high_queue_.size() + med_queue_.size() + low_queue_.size() >= max_capacity_) {
        if (shutdown_) return;
        if (!low_queue_.empty()) {
            low_queue_.pop(); // Drop oldest LOW event
            break;
        } else {
            not_full_cv_.wait(lock, [this]() {
                return shutdown_ || (size() < max_capacity_);
            });
            if (shutdown_) return;
        }
    }
    // ... enqueue by priority and notify workers
}

Deadlock-Free Dispatching

Subscribing happens during boot (rare writes), but lookups occur on every single event dispatch (constant concurrent reads).

We use a `std::shared_mutex` to allow threads to read callbacks concurrently. Critically, we copy the callback handles and release the read lock before executing them, avoiding deadlocks if a callback registers new handlers recursively.

// handler_registry.cpp
void HandlerRegistry::dispatch(const Event& event) const {
    std::vector handlers_to_call;
    {
        std::shared_lock lock(mutex_);
        auto it = registry_.find(event.type);
        if (it != registry_.end()) {
            handlers_to_call = it->second; // deep copy
        }
    } // Shared Lock released here!

    for (const auto& handler : handlers_to_call) {
        handler(event); // Executed outside lock
    }
}

Callback Exception Isolation

An unhandled exception in a background thread propagates to the root and triggers `std::terminate()`, crashing the entire application.

Our `ThreadPool` worker loops encapsulate subscriber execution inside isolated try-catch blocks. If a callback fails, the thread logs the error and continues looping safely instead of crashing.

// thread_pool.cpp
void ThreadPool::worker_loop() {
    while (running_ || !event_bus_.empty()) {
        try {
            Event event = event_bus_.pop();
            try {
                handler_registry_.dispatch(event);
            } catch (const std::exception& ex) {
                std::cerr << "[Exception] Callback failed: " << ex.what();
            } catch (...) {
                std::cerr << "[Exception] Unknown callback failure";
            }
        } catch (const std::exception&) {
            break; // Thrown by pop() on shutdown and dry queue
        }
    }
}

Drift-Free Scheduling Loop

Using relative sleeping (e.g. `sleep_for(16.6ms)`) introduces accumulative drift, because processing time inside the loop pushes scheduling targets backward.

We use absolute scheduling deadlines and sleep until them (`sleep_until`). This absorbs execution delay and maintains precise frequency grids.

// main.cpp (TickSource)
void TickSource::run() {
    auto start_time = steady_clock::now();
    auto next_input = start_time;
    const nanoseconds input_interval(1'000'000'000 / 60);

    while (running_) {
        auto now = steady_clock::now();
        if (now >= next_input) {
            push_event(EventType::INPUT, Priority::HIGH, ++event_id);
            next_input += input_interval; // Rigid addition
        }
        std::this_thread::sleep_until(next_wakeup);
    }
}
